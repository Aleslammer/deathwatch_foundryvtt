# Macro Development Pattern

## Overview

Macros use a two-file pattern to enable proper syntax highlighting and code editing while maintaining Foundry's required JSON pack format.

## File Structure

For each macro:

- **JavaScript source**: `src/packs-source/macros/macro-name.js`
  - Editable JavaScript with full syntax highlighting
  - Contains the actual macro code
  - Version controlled

- **JSON metadata**: `src/packs-source/macros/macro-name.json`
  - Foundry macro metadata (_id, name, type, img, etc.)
  - Empty `command` field (placeholder, regenerated by build)
  - Version controlled

## Development Workflow

### 1. Edit Macro Code

Edit the `.js` file directly with full JavaScript support:

```bash
# Open in your editor
code src/packs-source/macros/deathwatch-ranged-weapon-animation.js
```

Make your changes with syntax highlighting, IntelliSense, etc.

### 2. Compile Packs

Build the full pack system (merges `.js` files during compilation):

```bash
npm run build:packs
```

This runs:
1. `compactJson` - Format JSON files
2. `validatePacks` - Check for errors
3. `compilePacks` - Generate LevelDB packs (reads `.js` files and merges into pack)

**Important:** The compilation step reads `.js` files directly. Your source `.json` files remain clean (empty `command` field).

### 3. Deploy

Deploy to your local Foundry installation:

```bash
npm run build:copy
# or
npm run build:all  # Runs build:packs + build:copy
```

## Creating a New Macro

### Step 1: Create JSON Metadata

Create `src/packs-source/macros/my-macro.json`:

```json
{
  "_id": "macro-my-macro",
  "name": "My Macro Name",
  "type": "script",
  "img": "icons/svg/dice-target.svg",
  "flags": {},
  "folder": null,
  "sort": 0,
  "ownership": { "default": 0 },
  "author": "",
  "scope": "global",
  "command": "",
  "_stats": {
    "compendiumSource": null,
    "duplicateSource": null,
    "coreVersion": "13.0.0",
    "systemId": "deathwatch",
    "systemVersion": "0.0.2"
  }
}
```

**Important:**
- `_id` must be unique and match the pattern `macro-<name>`
- `type` must be `"script"`
- `command` should be empty string `""`
- `scope` can be `"global"` or `"actor"`

### Step 2: Create JavaScript Source

Create `src/packs-source/macros/my-macro.js`:

```javascript
/**
 * My Macro Name
 *
 * Description of what this macro does.
 */

// Your macro code here
console.log('Hello from my macro!');

// Access actor
const actor = game.user.character;
if (!actor) {
  ui.notifications.warn('No character selected');
  return;
}

// Do something with the actor
ui.notifications.info(`Selected: ${actor.name}`);
```

### Step 3: Build and Deploy

```bash
npm run build:all
```

## Compilation Details

**Location:** `builds/scripts/compilePacks.mjs`

**Macro Handling:**
1. Reads `macro-name.json` (metadata only, empty `command` field)
2. Checks for matching `macro-name.js` file
3. If `.js` exists, reads its contents
4. Merges JS into `command` field IN MEMORY
5. Writes merged data to LevelDB pack
6. Source `.json` file remains unchanged (clean)

**Result:** Source files stay clean in version control, compiled packs contain full JavaScript.

## Integration with Build Pipeline

```json
"build:packs": "node builds/scripts/compactJson.mjs && prettier --write --log-level warn \"src/packs-source/**/*.json\" && node builds/scripts/validatePacks.mjs && node builds/scripts/compilePacks.mjs"
```

Order:
1. **compactJson** - Compact/format JSON
2. **prettier** - Format JSON
3. **validatePacks** - Validate all pack data
4. **compilePacks** - Generate LevelDB packs (merges `.js` files in-memory)

## Important Notes

### Keep JSON `command` Field Empty

The `command` field in `.json` files MUST remain empty (`""`). The compilation process reads `.js` files and merges them in-memory during pack compilation.

✅ **Correct:** Edit `macro-name.js`  
✅ **Correct:** Keep `command: ""` in JSON  
❌ **Wrong:** Put code in `.json`'s `command` field

### Version Control

Both `.js` and `.json` files are version controlled:
- `.js` - Source of truth for code
- `.json` - Source of truth for metadata (with empty `command` field)

The JSON files remain clean in git. The compilation process merges them at build time only.

### Debugging

If a macro doesn't work after deployment:

1. **Check the JSON was updated:**
   ```bash
   node -e "const fs = require('fs'); const data = JSON.parse(fs.readFileSync('src/packs-source/macros/my-macro.json', 'utf8')); console.log('Command length:', data.command.length);"
   ```

2. **Check the pack was compiled:**
   ```bash
   npm run build:packs
   ```

3. **Check the pack was deployed:**
   ```bash
   npm run build:copy
   ```

4. **Restart Foundry** - Sometimes Foundry caches compendium data
