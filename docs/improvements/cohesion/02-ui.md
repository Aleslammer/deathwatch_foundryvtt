# Phase 2 — Cohesion UI

## Goal
Display the Kill-team's Cohesion pool as a persistent, always-visible floating panel that all players can see — independent of any character sheet.

## Design Decision: Foundry Application Window

Cohesion is a shared Kill-team resource, not per-actor data. Embedding it on every character sheet has problems:
- Duplicated on every open sheet
- Not visible when no sheet is open
- Awkward for a world-level value to live on an actor sheet

Instead, we create a **CohesionPanel** — a Foundry `Application` with `popOut: true` that renders as a proper floating window. It renders once on `ready`, stays visible at all times, and all clients see the same data via world settings.

## Implementation: `popOut: true`

We use `popOut: true` which gives us Foundry's built-in window management:
- Title bar with drag-to-reposition
- Renders into `#ui-windows` (proper floating layer)
- No custom drag code needed
- Position resets to top-center on each page load (no persistence — kept simple)
- `_render()` override positions window at top-center after each render

The `close()` method is overridden to prevent closing — the panel should always be visible. Only `close({ force: true })` can dismiss it.

## CohesionPanel (`src/module/ui/cohesion-panel.mjs`)

Extends `Application` (Foundry v13). Singleton pattern — one instance shared across the session.

```javascript
export class CohesionPanel extends Application {
  static _instance = null;

  static getInstance() {
    if (!CohesionPanel._instance) {
      CohesionPanel._instance = new CohesionPanel();
    }
    return CohesionPanel._instance;
  }

  static get defaultOptions() {
    return foundry.utils.mergeObject(super.defaultOptions, {
      id: 'cohesion-panel',
      title: '⚔ Kill-team Cohesion',
      template: 'systems/deathwatch/templates/ui/cohesion-panel.html',
      popOut: true,
      width: 220,
      height: 'auto',
      minimizable: false,
      resizable: false,
      classes: ['cohesion-panel']
    });
  }

  async close(options = {}) {
    if (options.force) return super.close(options);
    return; // Prevent closing
  }
}
```

## Panel Layout

Foundry window with title bar "⚔ Kill-team Cohesion", compact body:

```
┌─ ⚔ Kill-team Cohesion ─────────┐
│  ⚔ Cohesion         4 / 7      │  ← tooltip shows breakdown
│  Leader: Palarius               │
│  [+1] [−1] [⟳] [✎] [♛] [🎲]   │  ← GM only (except 🎲)
└─────────────────────────────────┘
```

| Button | Icon | Label | Who | Action |
|--------|------|-------|-----|--------|
| +1 | `fa-plus` | Recover | GM | `_adjustCohesion(1)` |
| −1 | `fa-minus` | Lose | GM | `_adjustCohesion(-1)` |
| ⟳ | `fa-sync` | Recalculate | GM | Opens recalculate dialog (with GM modifier) |
| ✎ | `fa-edit` | Edit | GM | Opens edit value/max dialog |
| ♛ | `fa-crown` | Set Leader | GM | Opens leader selection dialog |
| 🎲 | `fa-dice` | Challenge | All | Prompts actor selection, rolls Cohesion Challenge |

## Rendering & Lifecycle

### Initialization (`deathwatch.mjs`)

```javascript
// In Hooks.once('ready', ...)
CohesionPanel.getInstance().render(true);
```

### Reactivity

```javascript
Hooks.on('updateSetting', (setting) => {
  if (['deathwatch.cohesion', 'deathwatch.squadLeader', 'deathwatch.cohesionModifier'].includes(setting.key)) {
    CohesionPanel.getInstance().render(false);
  }
});
```

## Dialogs

### Recalculate — shows formula breakdown with editable GM modifier
### Edit — manual value/max override
### Set Leader — dropdown of all character actors, recalculates max on confirm
### Cohesion Challenge — auto-selects if player owns one character, otherwise prompts

## Settings (Phase 1 + 2 combined)

| Setting | Scope | Type | Purpose |
|---------|-------|------|---------|
| `deathwatch.cohesion` | world | Object | `{ value, max }` — shared pool |
| `deathwatch.squadLeader` | world | String | Actor ID of squad leader |
| `deathwatch.cohesionModifier` | world | Number | GM modifier to Cohesion max |
| `deathwatch.cohesionDamageThisRound` | world | Boolean | 1-per-round damage cap |
