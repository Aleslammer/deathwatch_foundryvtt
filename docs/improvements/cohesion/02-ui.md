# Phase 2 — Cohesion UI

## Goal
Display the Kill-team's Cohesion pool as a persistent, always-visible floating panel that all players can see — independent of any character sheet.

## Design Decision: Floating HUD Panel

Cohesion is a shared Kill-team resource, not per-actor data. Embedding it on every character sheet has problems:
- Duplicated on every open sheet
- Not visible when no sheet is open
- Awkward for a world-level value to live on an actor sheet

Instead, we create a **CohesionPanel** — a lightweight Foundry `Application` that floats over the UI (like the macro bar or token HUD). It renders once, stays visible at all times, and all clients see the same data via world settings.

## CohesionPanel (`src/module/ui/cohesion-panel.mjs`)

Extends `Application` (Foundry v13). Singleton pattern — one instance shared across the session.

```javascript
export class CohesionPanel extends Application {
  static _instance = null;

  static getInstance() {
    if (!CohesionPanel._instance) {
      CohesionPanel._instance = new CohesionPanel();
    }
    return CohesionPanel._instance;
  }

  static get defaultOptions() {
    return foundry.utils.mergeObject(super.defaultOptions, {
      id: 'cohesion-panel',
      template: 'systems/deathwatch/templates/ui/cohesion-panel.html',
      popOut: false,          // No window chrome — renders inline in the DOM
      classes: ['cohesion-panel']
    });
  }

  getData() {
    const cohesion = game.settings.get('deathwatch', 'cohesion');
    const leaderId = game.settings.get('deathwatch', 'squadLeader');
    const leader = leaderId ? game.actors.get(leaderId) : null;
    const gmMod = game.settings.get('deathwatch', 'cohesionModifier');
    return {
      value: cohesion.value,
      max: cohesion.max,
      leaderName: leader?.name || 'None',
      isGM: game.user.isGM,
      breakdown: CohesionHelper.buildCohesionBreakdown(leader, gmMod)
    };
  }

  activateListeners(html) {
    super.activateListeners(html);
    html.find('.cohesion-recover').click(() => CohesionHelper.recoverCohesion(1));
    html.find('.cohesion-lose').click(() => CohesionHelper.applyCohesionDamage(1));
    html.find('.cohesion-recalculate').click(() => this._onRecalculate());
    html.find('.cohesion-edit').click(() => this._onEdit());
    html.find('.cohesion-set-leader').click(() => this._onSetLeader());
    html.find('.cohesion-challenge-btn').click(() => this._onCohesionChallenge());
  }

  async _onRecalculate() { /* Recalculate dialog — same as before */ }
  async _onEdit() { /* Edit value/max dialog */ }
  async _onSetLeader() { /* Select leader from character actors */ }
  async _onCohesionChallenge() { /* Prompt for actor, roll 1d10 vs Cohesion */ }
}
```

### Why `popOut: false`?
A non-popOut Application renders its HTML directly into the DOM without Foundry's window chrome (title bar, close button, resize handles). This keeps the panel compact and HUD-like. We add lightweight drag support ourselves so users can reposition it freely.

### `popOut: false` vs `popOut: true` — Tradeoffs

We start with `popOut: false` for a minimal HUD look. This may change to `popOut: true` later if needed.

| Concern | `popOut: false` (current) | `popOut: true` |
|---------|--------------------------|----------------|
| Look & feel | Minimal, HUD-like, no chrome | Foundry window with title bar, close ✕, resize |
| Dragging | Custom implementation (mousedown/move/up) | Built-in via Foundry's window header drag |
| Position save | Custom client setting | Built-in via Foundry's `Application` position persistence |
| Close/reopen | Always visible (no close button) — desired behavior | User can close it; needs a way to reopen (scene control button or settings) |
| Z-order | Manual `z-index` in CSS | Foundry manages window stacking automatically |
| Accessibility | Must handle focus/keyboard manually if needed | Foundry's window management handles focus |
| Render target | Appends to `document.body` | Appends to `#ui-windows` (Foundry's window container) |

**When to switch to `popOut: true`:**
- If the custom drag implementation causes issues (e.g., conflicts with canvas interaction, mobile/touch problems)
- If we want Foundry's built-in position persistence instead of our custom client setting
- If users want to close/minimize the panel (would need a reopen button, e.g., scene control icon)

**If we switch**, the migration is straightforward:
1. Change `popOut: false` → `popOut: true` in `defaultOptions`
2. Add a `title` to `defaultOptions` (e.g., `'Kill-team Cohesion'`)
3. Remove custom drag code from `activateListeners()`
4. Remove `cohesionPanelPosition` client setting (Foundry handles it)
5. Remove `position: fixed` from CSS, adjust styling for window chrome
6. Add a scene control button or macro to reopen if closed

## Rendering & Lifecycle

### Initialization (`deathwatch.mjs`)

Render the panel once on the `ready` hook, after settings are registered:

```javascript
// In Hooks.once('ready', ...)
const cohesionPanel = CohesionPanel.getInstance();
cohesionPanel.render(true);
```

### Position Persistence (Client Setting)

Each user's panel position is stored as a **client-scoped** setting so it persists between sessions and each player can place it independently:

```javascript
// In init hook
game.settings.register('deathwatch', 'cohesionPanelPosition', {
  name: 'Cohesion Panel Position',
  scope: 'client',
  config: false,
  type: Object,
  default: { left: 15, bottom: 80 }   // Default: bottom-left above hotbar
});
```

### Position Restore

On render, apply the saved position:
```javascript
// In CohesionPanel, after render
_onRender(html) {
  const pos = game.settings.get('deathwatch', 'cohesionPanelPosition');
  const el = this.element[0] || this.element;
  el.style.left = `${pos.left}px`;
  el.style.bottom = `${pos.bottom}px`;
}
```

### Drag-to-Reposition

Lightweight drag handling on the panel element itself. Clicking and dragging the panel header/background repositions it. Clicking buttons does not trigger drag (events stop propagation).

```javascript
activateListeners(html) {
  super.activateListeners(html);
  // ... button listeners ...

  // Drag support
  const el = html[0] || html;
  let dragging = false, startX, startY, startLeft, startBottom;

  el.addEventListener('mousedown', (e) => {
    // Don't drag when clicking buttons/links
    if (e.target.closest('a, button, input, select')) return;
    dragging = true;
    el.classList.add('dragging');
    startX = e.clientX;
    startY = e.clientY;
    const rect = el.getBoundingClientRect();
    startLeft = rect.left;
    startBottom = window.innerHeight - rect.bottom;
    e.preventDefault();
  });

  window.addEventListener('mousemove', (e) => {
    if (!dragging) return;
    const dx = e.clientX - startX;
    const dy = e.clientY - startY;
    el.style.left = `${startLeft + dx}px`;
    el.style.bottom = `${startBottom - dy}px`;
  });

  window.addEventListener('mouseup', () => {
    if (!dragging) return;
    dragging = false;
    el.classList.remove('dragging');
    // Persist position
    const rect = el.getBoundingClientRect();
    game.settings.set('deathwatch', 'cohesionPanelPosition', {
      left: Math.round(rect.left),
      bottom: Math.round(window.innerHeight - rect.bottom)
    });
  });
}
```

### DOM Insertion

For a `popOut: false` Application, Foundry inserts the element into `document.body` by default. The CSS `position: fixed` ensures it appears at the correct screen location regardless of where it sits in the DOM.

## Reactivity

When any client updates the cohesion or squad leader settings, re-render the panel:

```javascript
// In deathwatch.mjs
Hooks.on('updateSetting', (setting) => {
  if (['deathwatch.cohesion', 'deathwatch.squadLeader', 'deathwatch.cohesionModifier'].includes(setting.key)) {
    CohesionPanel.getInstance().render(false);
  }
});
```

This replaces the previous plan to re-render all actor sheets. The panel is the single source of truth for Cohesion display.

## Recalculate Dialog

Same as before — shows formula breakdown with editable GM modifier:

```javascript
async _onRecalculate() {
  const leaderId = game.settings.get('deathwatch', 'squadLeader');
  const leader = leaderId ? game.actors.get(leaderId) : null;
  if (!leader) return ui.notifications.warn('No squad leader assigned.');

  const currentGmMod = game.settings.get('deathwatch', 'cohesionModifier');
  const fsBonus = Math.floor((leader.system.characteristics?.fs?.value || 0) / 10);
  const rankMod = CohesionHelper.getRankModifier(leader.system.rank || 1);
  const commandMod = CohesionHelper.getCommandModifier(leader.system.skills?.command || {});

  new Dialog({
    title: 'Recalculate Cohesion',
    content: `
      <div class="form-group">
        <p>Fellowship Bonus: <strong>${fsBonus}</strong></p>
        <p>Rank Modifier: <strong>+${rankMod}</strong></p>
        <p>Command Modifier: <strong>+${commandMod}</strong></p>
        <hr/>
        <label>GM Modifier:</label>
        <input type="number" id="gm-modifier" value="${currentGmMod}" />
      </div>`,
    buttons: {
      confirm: {
        label: 'Recalculate',
        callback: async (html) => {
          const gmMod = parseInt(html.find('#gm-modifier').val()) || 0;
          await game.settings.set('deathwatch', 'cohesionModifier', gmMod);
          const max = CohesionHelper.calculateCohesionMaxFromActor(leader, gmMod);
          const current = game.settings.get('deathwatch', 'cohesion');
          await game.settings.set('deathwatch', 'cohesion', { value: Math.min(current.value, max), max });
        }
      },
      cancel: { label: 'Cancel' }
    },
    default: 'confirm'
  }).render(true);
}
```

## Set Squad Leader Dialog

```javascript
async _onSetLeader() {
  const characters = game.actors.filter(a => a.type === 'character');
  if (!characters.length) return ui.notifications.warn('No character actors found.');

  const options = characters.map(a => `<option value="${a.id}">${a.name}</option>`).join('');
  const currentLeader = game.settings.get('deathwatch', 'squadLeader');

  new Dialog({
    title: 'Set Squad Leader',
    content: `
      <div class="form-group">
        <label>Squad Leader:</label>
        <select id="leader-select">${options}</select>
      </div>`,
    render: (html) => { html.find('#leader-select').val(currentLeader); },
    buttons: {
      confirm: {
        label: 'Confirm',
        callback: async (html) => {
          const leaderId = html.find('#leader-select').val();
          await game.settings.set('deathwatch', 'squadLeader', leaderId);
          const gmMod = game.settings.get('deathwatch', 'cohesionModifier');
          const leader = game.actors.get(leaderId);
          const max = CohesionHelper.calculateCohesionMaxFromActor(leader, gmMod);
          await game.settings.set('deathwatch', 'cohesion', { value: max, max });
        }
      },
      cancel: { label: 'Cancel' }
    },
    default: 'confirm'
  }).render(true);
}
```

## Edit Cohesion Dialog

```javascript
async _onEdit() {
  const current = game.settings.get('deathwatch', 'cohesion');
  new Dialog({
    title: 'Edit Cohesion',
    content: `

      },
      cancel: { label: 'Cancel' }
    },
    default: 'confirm'
  }).render(true);
}
```
